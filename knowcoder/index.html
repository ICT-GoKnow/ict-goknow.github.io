Abstract

Adopting Large Language Models (LLMs) to conduct Universal Information Extraction(UIE) faces two main challenges, namely, the lack of a unified schema representation method that LLMs can easily understand and an effective learning framework that encourages LLMs to accurately follow specific schemas for extracting structured knowledge. For the first challenge, we introduce a code-style schema representation method to uniformly transform different schemas predefined for the UIE task into Python classes, with which the UIE task can be converted to a code generation process so as better to leverage LLMs' inherent code comprehension and generation abilities. We further construct a comprehensive code-style schema library covering over 30,000 types of knowledge, including entities, relations, and events, which is the largest one for UIE, to the best of our knowledge. To overcome the second challenge, we design a learning framework to ease the training of the LLM in a two-phase manner which enhances its schema understanding and schema following abilities, respectively. After training on large-scale automatically con- structed data, we obtain a powerful LLM called KnowCoder. Additionally, based on our unified schema representation, various IE datasets can be simultaneously utilized to refine the model. Experimental results show that KnowCoder achieves average relative improvements up to 2.9%, 19.1%, and 8.6% under the zero-shot, low-resource, and supervised settings, respectively.

Overview

We release KnowCoder, a powerful Large Language Model training on automatically constructed data, experimentally demonstrates strong generalization ability on unseen schemas and achieves average improvements up to 2.9%, 19.1% under the zero-shot and low-resource settings, respectively.

KnowCoder Schema

Code-style Schema Representation Method

The code-style schema representation method comprises three basic classes, namely, "Entity", "Relation", and "Event". Based on the three basic classes, we represent all the concepts in the schemas by the corresponding classes. Thus, the instances of each concept can be represented by the objects of the corresponding class. As shown in A schema consists of class name, class inheritance, class comments, type hint, and class method. The detailed explanation of each component can be found in the Section 2.1 in the paper.

Schema Library Construction

We construct the code-style schema library under this schema representation method based on Wikidata (Note that we use the Wikidata dump up to 20220704). We select the concepts included in the existing IE datasets created from Wikidata, i.e., KELM, UniversalNER, InstructIE, and LSEE, and derive the constraints among concepts according to their co-occurrences. To construct the taxonomies, we extract the "subclass of" relations among these concepts from Wikidata. To obtain the description of a concept, we use its definition from Wikidata directly or generate its descriptions using GPT-4 if its definition in Wikidata is missing. Finally, the constructed schema library encompasses over 29,177 entity types, 876 relation types, and 519 event types. The detailed statistics of the schema are shown in Table \ref{schema_stat}. Here, "#Type" denotes the total number of types, "#Type w/ desc." indicates the count of types with descriptions, and "#Type w/o desc." signifies the count of types without descriptions.

Specific Domain IE Data

The specific domain IE data can be found in 🤗Huggingface-KnowCoder-Data-specific-domain-IE-data.

We conducted experiments using 33 specific domain Information Extraction (IE) datasets, including 23 datasets for Named Entity Extraction (NER), 8 datasets for Relation Extraction(RE), 2 datasets for Event Detection (ED) task, and Event Argument Extraction (EAE) task. For NER task, we employed 23 datasets, including ACE 2004, ACE 2005, AnatEM, Broad Twitter, bc2gm, bc5cdr, CoNLL 2003, DIANN, FabNER, FindVehicle, GENIA, MIT Movie, MIT Restaurant, MultiNERD, ncbi-disease, Ontonotes5, WikiANN, WNUT 2017 and five CrossNER subsets (AI, literature, music, politics, science).

For the RE task, we use 8 datasets, including ACE 2005, ADE corpus, CoNLL 2004, GIDS, kbp37, NYT.

For the EE task, ACE 2005, CASIE are used.
